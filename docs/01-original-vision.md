# BoxingCat（拳击猫）— 原始愿景阐述

> **本文件完整保留创始人的原始阐述，不做任何删减、改写或"优化"。**
> 后续所有产品设计和技术决策均以此为准。

---

## 一、产品定位

桌面宠物，名为 **BoxingCat（拳击猫）**。

目标用户：**普通宅男程序员、二次元群体**——这些人在电脑前久坐，需要一个有吸引力、有养成感的桌面宠物来驱动他们做运动。

---

## 二、账户体系

1. **主人账户（Master Account）**
   - 第一次使用需要注册主人账户
   - 这是链上账户（"恋上账户"——推测为区块链钱包账户）

2. **Agent 账户（猫的账户）**
   - 购买 NFT 之前需要先注册主人账户
   - 注册主人账户后，为猫猫生成它的 Agent 账户

3. **NFT 体系**
   - 猫的具体表现形态（样子）需要通过购买 NFT 来获得
   - NFT 是基础门槛——必须先有 NFT 才有猫的外观

---

## 三、核心玩法一：拳击健身（日常核心循环）

### 3.1 基本流程

1. 用户坐在电脑前
2. 打开摄像头
3. 做拳击动作
4. AI 摄像头识别动作
5. 系统判别拳击姿势是否正确、到不到位
6. 根据每组动作的量（次数/质量）给予评分
7. 评分转化为 **猫粮** 奖励

### 3.2 锻炼效果

> "其实如果做专业的话，是能够达到一定的锻炼效果的"

- 拳击本身是有氧运动
- 通过 AI 判别姿势正确性，确保动作标准
- 量化训练量（组数、评分）

### 3.3 与猫的关联

- 锻炼成果直接影响猫的 **敏捷度（Agility）**
- 敏捷度 = 拳击训练次数 + 评分评估的综合结果
- 敏捷度在后续"飞盘"玩法中起关键作用

---

## 四、核心玩法二：猫猫出门（旅行系统）

### 4.1 设计参考

> "他还会出门，跟以前一个游戏叫**旅行的青蛙**类似"

- 参考《旅行的青蛙》（Tabikaeru / Travel Frog）
- 猫猫会自主出门旅行
- 旅行中发生随机事件
- 带回来新的消息/见闻/物品

### 4.2 特点

- 异步体验——猫出门了，你不知道它什么时候回来
- 带回的东西有随机性
- 增加养成感和期待感

---

## 五、核心玩法三：飞盘社交

### 5.1 购买飞盘

- 飞盘是一种道具
- 购买飞盘需要消耗 **猫粮**

### 5.2 飞盘飞行

- 飞盘会随机飞到别人家（其他用户的桌面）
- 相当于一种异步社交/互动机制

### 5.3 捕获飞盘（别人家）

当飞盘飞到别人家时，有两种可能的结果：

#### 结果 A：捕获成功

- 前提：接收方的猫**敏捷度足够高**
- 敏捷度来源于对方的拳击锻炼成果
- 捕获成功后：
  - 可以**卖掉飞盘**换猫粮
  - 也可以**自己玩**
  - 都能获得猫粮收益

#### 结果 B：捕获失败（飞走）

- 猫的敏捷度不够
- 飞盘飞走
- **但飞盘不仅仅是飞走**——它会路过时往你家桌子上 **丢猫屎**
- 猫屎需要花费 **猫粮** 去打扫
- "打扫就需要花钱"——这是失败惩罚机制

### 5.4 社交飞盘的经济循环

```
购买飞盘（消耗猫粮）
  → 飞盘随机飞到别人家
    → 对方捕获成功 → 对方获得猫粮（卖/玩）→ 对方锻炼更多
    → 对方捕获失败 → 对方被丢猫屎 → 对方花猫粮打扫（惩罚）
```

## 六、核心玩法四：广场（远期规划）

### 6.1 广场功能

- 任务系统
- 兑换系统
- 共识和协商机制

### 6.2 社区接入

> "广场是为更多的社区服务，接入各个社区"

- 这是后续发展阶段
- 社区驱动的内容和交互

---

## 七、NFT 与外观系统

1. 猫的外观/样子由 NFT 决定
2. 购买 NFT 是获得猫外观的前提
3. NFT 可能有不同的稀有度/样式
4. NFT 与链上账户绑定

---

## 八、技术需求初步描述

1. **桌面宠物渲染**——猫需要在桌面上显示（可能为透明窗口叠加层）
2. **AI 动作识别**——摄像头捕捉 + AI 模型判别拳击姿势
3. **区块链/NFT**——主人账户、Agent 账户、NFT 购买与绑定
4. **实时评分系统**——基于 AI 识别结果对每组动作评分
5. **飞盘动画**——飞盘的飞行、捕获、猫屎等动画效果
6. **异步事件系统**——猫出门旅行、飞盘飞来等异步事件
